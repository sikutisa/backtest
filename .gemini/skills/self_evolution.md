# Skill: Self-Evolution Protocol (SEP)

This skill governs the continuous improvement of the workspace's autonomous capabilities by objectifying recurring workflows.

## 🔄 Evolution SOP
1. **Identify Patterns**: Recognize repetitive or complex workflows (e.g., scaffolding a new module).
2. **Objectify Knowledge**: Create a new Standard Operating Procedure (SOP) under `.gemini/skills/`.
3. **Refactor & Modularize**: Move complex logic from `AGENTS.md` or general prompts into specialized skill files.
4. **Context Review**: Regularly audit `AGENTS.md` and `GEMINI.md` to ensure they reflect the current architectural state.
5. **Expert Registry**: Update the project's documentation when a new professional domain is mastered.

## 📜 Execution Rules
- When a new skill is created, immediately inform the user and explain its utility.
- Always prioritize using an existing skill over reinventing a workflow.
